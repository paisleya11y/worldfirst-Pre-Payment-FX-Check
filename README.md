# WorldFirst AI 资金助手 — 面试 Demo

> 一张「换汇前资金建议卡」：在商家点击"兑换"之前的 10 秒，AI 帮他想清楚 ——
> **这笔钱换完要做什么、该换多少、要不要先补凭证。**

```bash
cd /Users/linpeixuan/worldfirst
```

## 面试演示动线（90 秒主链路）

### Step 1 — 进入 WorldFirst 后台
打开 `localhost:5173/`，对应**首页**。

讲故事：「这是 WorldFirst 后台，我是一个跨境电商商家，月 GMV 约 50 万美元，多平台经营。」

### Step 2 — 进入货币兑换页
点击侧边栏「资金管理」或快捷导航的「货币兑换」（带 AI 红色 badge）。

### Step 3 — 输入兑换金额触发 AI 卡
在金额输入框输入 `50000`（或双击输入框自动填入），失焦 600ms 后 **AI 资金建议卡淡入**。

> **演示要点**：AI 不弹问卷，它**已经看懂用户**了。
> 卡片显示：
> - 预计用途：**保留外币付款**
> - 依据：你近 30 天有 5 笔 USD 付款记录…

### Step 4 — 点「查看建议方案」展开
卡片展开三步建议 + **路径对比**（当前路径 vs 建议路径 vs 差异）。

> **演示要点**：路径对比用「**减少路径成本**」措辞——不写"多赚"。
> 这是合规设计：AI 不做投资建议。

### Step 5 — 点「追问 AI」打开对话
右侧抽屉滑出，已经把当前情境注入到上下文。

可以点预设 chip，也可以自由提问。**这一段接的是真 DeepSeek API**。

试着问：「你觉得明天美元会涨还是跌？」——AI 会守红线，说"无法预测"。

### Step 6 — 点「一键应用建议」
- 兑换金额自动改成 20,000（不替用户付，是预填）
- 弹出 toast："已应用：兑换 20,000 USD + 保留 30,000 USD + 设置汇率提醒"
- 兑换页底部出现绿色 banner："减少路径成本约 ¥XXX"

## 切换其他场景演示（可选）

页面**左下角**有个 demo helper 工具条，4 个按钮：

| 按钮 | 演示什么 |
|---|---|
| **保留外币** | 主舞台，主动建议（介入强度 B） |
| **付供应商** | **静默校验**，AI 只显示一行轻提示，不打扰（介入强度 A） |
| **暂时不用** | 主动建议，但克制（不推安心兑/万余盈） |
| **额度不足** | **异常拦截**（介入强度 C），跳补凭证（最体现懂监管） |

> 也可以点卡片里的「可修改 ▾」切换用途，效果一样。

**讲故事顺序**：保留外币（最华丽）→ 付供应商（最克制）→ 额度不足（最合规）。

三种 AI 介入强度一起看才完整。

## 关键产品设计（被问到时讲）

### 三不原则
1. **不预测汇率**——卡片只引用历史区间，不说"将上涨"
2. **不计算金额**——所有数字由后台规则引擎输出，LLM 只做话术组装
3. **不自动执行**——所有动作必须用户二次确认

### LLM 在哪、不在哪
- **在**：追问对话区（解释为什么、根据用户新信息改建议）
- **不在**：金额计算、用途识别、推荐方案——这些都是规则引擎做的

> 这条很关键：演示里"展开建议"那部分是脚本化的（稳定），追问对话是真 LLM（灵活）。
> 这正好对应 PRD 里的"规则引擎决定数字、LLM 决定话术"。

### 北极星：有效决策完成率
**不是**「AI 卡片采纳率」（那会诱导强推）。
而是用户在卡片展示后**完成任一与建议一致的低风险动作**：
- 按建议部分兑换 / 保留外币 / 设置汇率提醒 / 跳补凭证

反向兜底硬约束：错误推荐率 < 3% / 取消率 < 8% / 关闭率 < 50%

### 商业飞轮
AI 推荐越准 → 商家更敢在 WorldFirst 体内动钱 → FX spread + AUM + 通道费 + 卡 interchange ↑ → 数据回流让 AI 更准。

## 架构

```
浏览器
  ├─ index.html / exchange.html         (现网视觉对齐)
  ├─ styles.css                          (设计 token 全套)
  └─ app.js                              (4 个场景脚本化数据 + 交互)
                                                ↓
                                          (fetch /api/chat)
                                                ↓
node server.js  (单文件)
  ├─ 静态托管
  └─ /api/chat → DeepSeek API
                  + System Prompt (注入 PRD 三不原则)
                  + 上下文（当前场景、金额、建议、依据）
```

API key 在 `server.js` 里，**不暴露到前端**。

如果 DeepSeek API 挂了（网络问题），追问区会自动 fallback 到本地预设回答，演示不会断。
